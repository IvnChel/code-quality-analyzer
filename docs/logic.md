# Логика оценки качества кода

Этот проект анализирует файлы Python по нескольким метрикам. Итоговая оценка (максимум 15 баллов) формируется следующим образом:

## Основные метрики:
1. **Наличие Docstrings**: За каждую функцию или класс с описанием начисляются баллы.
2. **Сложность (Complexity)**: Анализатор проверяет длину функций и количество логических переходов.
3. **Стиль (Naming)**: Проверка на соответствие стандартам PEP8 (snake_case для функций, PascalCase для классов).
4. **Дубликаты**: Поиск повторяющихся строк кода, которые стоит вынести в отдельные методы.

## Как запустить:
1. Установите зависимости: `pip install -r requirements.txt`
2. Запустите анализ: `python src/main.py`
3. Результат будет выведен в консоль и сохранен в `full_report.txt`.
